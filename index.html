Project Documentation

Image Meta Data Filter

A backend service that authenticates users, accepts image uploads, extracts EXIF metadata, stores records in MongoDB, and returns filterable image collections.

Node + Express MongoDB + Mongoose JWT Authentication EXIF Metadata Extraction

1) What It Does For Users

Secure Sign Up and Login

Users register and log in to receive JWT tokens, so every image action is scoped to the correct account.

Upload and Auto-Read Metadata

When users upload JPEG, PNG, or TIFF files, EXIF metadata like camera, capture date, and dimensions is extracted automatically.

Find Images Faster

Users can filter by year, camera, and location fields and sort by newest, oldest, metadata date, or filename.

2) Folder, File, and Package Map

How each folder contributes to the app and which package powers it.

Core App Layer

  • index.js: server bootstrap, middleware setup, static uploads route, API route mounting, 404 and global error handling.
  • config/db.js: MongoDB connection using mongoose.
  • config/validateEnv.js: startup guard that verifies required environment variables.

Security and Validation Middleware

  • middleware/ratelimiter.js: request throttling for general and auth-heavy routes.
  • middleware/requireAuth.js: checks Bearer token, verifies JWT, injects user claims into request.
  • middleware/validate.js: request-body validation for register/login with express-validator.

Business Routes and Models

  • routes/auth.js: register/login endpoints, password hashing, JWT generation.
  • routes/images.js: upload, list, view, update, and delete image records with ownership checks.
  • models/User.js and models/Image.js: MongoDB schemas for users and image metadata documents.

Storage and Infra Files

  • uploads/: physical directory where multer stores image files.
  • package.json: scripts and dependency catalog for runtime and development.
  • docker-compose.yml: container orchestration entry point for local stack setup.

Packages Used and Why

express (API framework), mongoose (MongoDB ORM), jsonwebtoken (auth tokens), bcryptjs (password hashing), dotenv (env loading), multer (uploads), exif-parser (metadata extraction), cors (cross-origin policy), helmet (security headers), express-rate-limit (abuse protection), express-validator (input validation), nodemon (dev reload).

3) End-to-End Workflow

Which file provides what logic, and how requests flow through the project.

  1. 1. Boot: index.js loads env vars, validates required keys, then connects to MongoDB.
  2. 2. Global Guards: index.js attaches rate limiting, helmet, CORS, and JSON parsing middleware.
  3. 3. Auth Entry: routes/auth.js handles register/login with validate.js checks, bcryptjs hashing, and JWT token creation.
  4. 4. Protected Access: routes/images.js requires middleware/requireAuth.js to verify Bearer tokens and bind user identity.
  5. 5. Upload Pipeline: multer stores files in uploads/, exif-parser reads metadata, and models/Image.js persists records to MongoDB.
  6. 6. Query and Manage: image list endpoint applies user scope, filters, sorting, and pagination; update/delete enforce ownership checks.
  7. 7. Error Paths: unknown routes return 404 and unexpected exceptions are routed to the global error handler.
